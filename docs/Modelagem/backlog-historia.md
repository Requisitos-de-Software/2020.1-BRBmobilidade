## Product Backlog
O Product Backlog é uma lista contendo todas as funcionalidades desejadas para um produto. O Product Backlog não precisa estar completo no início de um projeto. O conteúdo desta lista é definido pelo Product Owner. Pode-se começar com tudo aquilo que é mais óbvio em um primeiro momento. Com o tempo, o Product Backlog cresce e muda à medida que se aprende mais sobre o produto e seus usuários.

## História de Usuário
Uma história de usuário é uma ferramenta usada no desenvolvimento de software Ágil para capturar uma descrição de um recurso de software da perspectiva do usuário final.

## Lista das Histórias de usuário
### Épico 1: Acesso a aplicação

US01 - Eu como um usuário do sistema e proprietário do Bilhete Único desejo realizar o login
Critérios de aceitação:

* Deve conter botão de realizar login na aplicação.
* Deve possuir áreas para preencher dados de usuário e senha.
* Deve se conectar com o servidor do BRB para mostrar informações do usuário.
* Deve realizar o login

Requisitos relacionado a história: 

* RF09 via observação.

### Épico 2: Área do usuário

US03 - Eu como um usuário desejo ter um menu principal com as principais informações do sistema
Critérios de aceitação:

* Deve mostrar informações pertinentes ao usuário como saldo e acessos rápidos a outros menus
* Deve ser simples mas informativo sem poluir a tela.
* Deve atualizar com alterações de linhas favoritas e recargas de saldo.

Requisitos relacionado a história: 

* RF06 via observação.
* RF13 via observação.
* RF12 via observação.
* RNF05 via observação.

US04 - Eu como um usuário desejo ter um menu para as linhas de ônibus
Critérios de aceitação:

* Deve mostrar as linhas favoritas do usuário para acesso mais rápido.
* Deve disponibilizar sistema de busca que filtra por linhas ou destinos.

Requisitos relacionado a história: 

* RF06 via observação.
* RF14 via observação.
* RNF04 via observação.

US05 - Eu como um usuário desejo acessar a área do estudante.
Critérios de aceitação:

* Deve redirecionar o usuário a área específica com informações de cartões estudantis.

Requisitos relacionado a história: 

* RF01 via observação.

US06 - Eu como um usuário desejo saber informações detalhadas de horários e ônibus
Critérios de aceitação:

* Deve listar quais estações o ônibus específico para.
* Deve mostrar a rotina de horários completa da linha específica.
* Deve possibilitar o usuário de favoritar a linha.
* Deve disponibilizar um mapa com o trajeto do ônibus.

Requisitos relacionado a história: 

* RF06 via observação.
* RF14 via observação.

US07 - Eu como um usuário desejo visualizar o saldo do cartão
Critérios de aceitação:

* Deve mostrar ao usuário qual o saldo do cartão na tela inicial.
* Deve informar o horário e dia da última atualização de saldo para evitar que o usuário receba a informação errada.

Requisitos relacionado a história: 

* RF10 via observação.

US08 - Eu como um usuário desejo visualizar os pontos de recarga
Critérios de aceitação:

* Deve disponibilizar um botão no menu para acessar os pontos de recarga.
* Deve listar todos os pontos de recarga no DF e possibilidade de acessar informações específicas de cada ponto.
* Deve informar endereço, horário de funcionamento e telefone de cada ponto específico.
* Deve mostrar no mapa a localização de cada ponto de recarga.

Requisitos relacionado a história: 

* RF05 via observação.
* RF03 via instropecção.

US09 - Eu como um usuário desejo trocar de cartão.
Critérios de aceitação:

* Deve disponibilizar um botão na homepage para trocar de cartão do usuário.
* Deve mostrar o nome de cada cartão para melhor navegação do usuário.

Requisitos relacionado a história: 

* RF08 via observação.
* RNF05 via observação.

US10 - Eu como um usuário desejo recarregar o meu cartão
Critérios de aceitação:

* Deve disponibilizar no menu principal e na homepage um botão para recarregar o cartão.
* Deve perguntar ao usuário o quanto ele deseja recarregar dentro dos limites mínimos(5 reais) e máximos(1000 reais).
* Deve gerar um boleto para o usuário recarregar o cartão.

Requisitos relacionado a história: 

* RF14 via observação.

US11 - Eu como usuário desejo acessar o meu extrato de uso
Critérios de aceitação:

* Deve disponibilizar no menu principal e na homepage um botão para acessar o extrato de uso.
* Deve mostrar cada recarga e uso do cartão em até um ano.

Requisitos relacionado a história: 

* RF13 via observação.
